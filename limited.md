# 已知限制

本文档记录当前最小闭环版本明确保留的限制。

## 身份与权限

- 当前已接入最小登录态，但账号仍是固定开发账号
- 服务端现在通过 Bearer token 识别当前用户，但会话保存在进程内存中，服务重启后需要重新登录
- 当前认证方案只适合开发和最小闭环验证，不适合正式上线

## 前端使用方式

- 前端已支持运行时登录，但仍只支持固定开发账号 `user_a` 和 `user_b`
- 前端 API 地址已完成方案二收口，统一从前端配置模块读取；当前默认值仍为 `http://127.0.0.1:3000`

## 功能范围

- 不支持通知
- 不支持分页加载更多
- 不支持 30 天自动清理
- 不支持会话列表
- 不支持已读回执
- 不支持撤回

## 从 Min Loop 到完整版仍需实现

- 登录态与真实身份认证
- 多用户运行时切换，而不是前端固定用户常量
- 会话列表页
- 图片发送增强，包括对象存储、缩略图、压缩策略和更稳的失败重试
- 表情发送增强，包括更完整的表情面板、消息编码方式和输入体验
- 通知能力
- 历史消息分页加载
- 30 天自动清理任务
- 已读回执
- 撤回消息
- 更完整的输入校验与风控
- 更严格的消息与查询权限控制
- 局域网或线上部署配置，而不是固定本机地址
- 持久化会话与更安全的认证存储方案
- 密码管理、找回、修改与账号生命周期
