# 已知限制

本文档记录当前最小闭环版本明确保留的限制。

## 身份与权限

- 当前已接入最小登录态，但账号仍是固定开发账号
- 服务端现在通过 Bearer token 识别当前用户，但会话保存在进程内存中，服务重启后需要重新登录
- 当前认证方案只适合开发和最小闭环验证，不适合正式上线

## 前端使用方式

- 前端已支持运行时登录，但仍只支持固定开发账号 `user_a` 和 `user_b`
- 前端 API 地址已支持运行时注入和按当前访问主机自动推导
- 当前仍未切到相对路径 `/api` 和开发代理 / 反向代理方案
- 局域网访问时通常仍需要显式设置 `SERVER_HOST`，跨源场景可能还需要设置 `CORS_ALLOW_ORIGIN`

## 功能范围

- 不支持会话列表
- 不支持已读回执
- 不支持撤回

- 登录态与真实身份认证
- 多用户运行时切换，而不是前端固定用户常量

- 表情发送增强，包括自定义表情包、表情搜索和与文本输入更深的融合体验
- 浏览器系统通知、离线推送和更完整的通知偏好设置
- 历史消息分页增强，例如自动触顶加载和更稳的滚动位置恢复
- 30 天自动清理增强，例如独立调度器、归档策略和清理监控
- 已读回执
- 撤回消息
- 更完整的输入校验与风控
- 更严格的消息与查询权限控制
- 相对路径 `/api`、开发代理与同源部署收敛
