# 更新项

本文档记录当前必须优先修复的问题，以及本轮已经落地的修复。

## 今天必须修

- [x] 轮询只允许把当前会话的新消息追加到当前聊天页，避免串入其他会话消息
- [x] 修复 `poll` 使用 `serverTime` 时的时间窗口漏消息问题
- [x] 发送消息前校验发送方是否已建立绑定
- [x] 为关键输入和请求体补基础长度上限

## 本轮已完成

### 1. 轮询只收当前会话消息

- 前端轮询结果追加前，已按当前 `targetUserId` 对应的 `conversationId` 过滤
- 不属于当前聊天页的消息，不再混入当前消息列表

### 2. 修复轮询漏消息窗口

- 后端 `poll` 现在会在查询开始时记录一个固定的 `serverTime`
- 查询只返回 `sent_at > since` 且 `sent_at <= serverTime` 的消息
- 下一次轮询继续使用这个固定 `serverTime`，避免“查询后到返回前”落库的消息被跳过

### 3. 发送前绑定校验

- 已要求发送方先建立 `senderId -> receiverId` 绑定
- 未绑定时，`POST /messages/text` 会直接拒绝发送

### 4. 输入与请求体长度上限

- `userId` 相关字段最大长度为 64
- `text` 最大长度为 1000
- `GET /messages/conversation` 的 `limit` 最大为 100
- JSON 请求体最大为 16 KB
